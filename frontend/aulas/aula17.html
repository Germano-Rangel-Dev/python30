Dia 17 – Tratamento de Erros em Python

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

Em programas reais, erros acontecem: entradas inválidas, arquivos inexistentes, divisões por zero, falhas de conversão, entre outros. Se esses erros não forem tratados, o programa pode encerrar de forma abrupta, prejudicando a experiência do usuário.

O tratamento de erros permite que o programa lide com situações inesperadas de forma controlada, exibindo mensagens claras e mantendo a execução estável. Em Python, isso é feito principalmente com try e except.

Conceito-chave:
Tratamento de erros permite capturar exceções e evitar que o programa falhe de forma inesperada.

2️⃣ Explicação Técnica Detalhada

O bloco try envolve o código que pode gerar um erro. Caso uma exceção ocorra, a execução salta para o bloco except.

É possível tratar exceções específicas (como ValueError, ZeroDivisionError) ou usar um except genérico. Também podemos usar else (executa se não houver erro) e finally (executa sempre).

3️⃣ Exemplos Práticos (do básico ao avançado)

Exemplo 1 – Tratamento básico


try:
    numero = int(input("Digite um número: "))
    print("Número digitado:", numero)
except ValueError:
    print("Erro: você não digitou um número válido.")

Captura erro de conversão quando o usuário digita algo que não é número.

Exemplo 2 – Divisão por zero


try:
    a = int(input("Digite o numerador: "))
    b = int(input("Digite o denominador: "))
    print(a / b)
except ZeroDivisionError:
    print("Erro: divisão por zero não é permitida.")

Evita que o programa quebre ao tentar dividir por zero.

Exemplo 3 – try / except / else / finally


try:
    x = int(input("Digite um número: "))
    resultado = 10 / x
except Exception as erro:
    print("Erro:", erro)
else:
    print("Resultado:", resultado)
finally:
    print("Fim da execução")

Demonstra o fluxo completo de tratamento de exceções.

    print("Erro de entrada ou divisão.")
