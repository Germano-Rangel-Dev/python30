Dia 19 – Encapsulamento, Herança e Polimorfismo

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

No dia anterior, você aprendeu a criar classes e objetos. Agora é o momento de aprofundar a Programação Orientada a Objetos entendendo como proteger dados, reaproveitar código e permitir comportamentos flexíveis.

Esses objetivos são alcançados através de três pilares fundamentais da POO: Encapsulamento, Herança e Polimorfismo.

Conceito-chave:
Encapsulamento protege dados, herança reaproveita código e polimorfismo permite comportamentos diferentes para métodos com o mesmo nome.

2️⃣ Explicação Técnica Detalhada

🔒 Encapsulamento

Encapsulamento consiste em restringir o acesso direto aos atributos de um objeto. Em Python, isso é feito por convenção usando um sublinhado _ ou dois sublinhados __.

🧬 Herança

Herança permite criar uma nova classe baseada em outra, reaproveitando atributos e métodos existentes.

🔁 Polimorfismo

Polimorfismo permite que métodos com o mesmo nome tenham comportamentos diferentes dependendo da classe.

3️⃣ Exemplos Práticos (do básico ao avançado)

Exemplo 1 – Encapsulamento


class Conta:
    def __init__(self, saldo):
        self._saldo = saldo

    def mostrar_saldo(self):
        return self._saldo

O atributo _saldo não deve ser acessado diretamente.

Exemplo 2 – Herança


class Animal:
    def falar(self):
        print("O animal faz um som")

class Cachorro(Animal):
    def falar(self):
        print("O cachorro late")

A classe Cachorro herda de Animal.

Exemplo 3 – Polimorfismo


animais = [Animal(), Cachorro()]

for animal in animais:
    animal.falar()

Cada objeto responde ao método falar() de forma diferente.
