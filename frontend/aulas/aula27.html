Dia 27 – Deploy de Aplicações Python

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

Criar uma aplicação funcionando localmente é apenas o começo. Para que outras pessoas possam acessar seu sistema, ele precisa estar disponível na internet. Esse processo é chamado de deploy.

Deploy é o ato de publicar sua aplicação em um servidor, permitindo acesso remoto, escalabilidade e uso real em produção.

Conceito-chave:
Deploy é o processo de disponibilizar uma aplicação para uso público.

2️⃣ Tipos de Deploy

  • Local – Apenas para testes
  • Servidor VPS – Controle total
  • Plataformas Cloud – Simplicidade e escala

Neste curso, focamos no conceito e nos passos universais, que funcionam em qualquer plataforma moderna.

3️⃣ Preparando o Projeto para Deploy

Passo 1 – Arquivo requirements.txt


fastapi
uvicorn
python-jose
passlib[bcrypt]

Passo 2 – Estrutura do projeto


backend/
 ├── main.py
 ├── database.py
 ├── auth.py
 └── requirements.txt

4️⃣ Executando em Produção


uvicorn main:app --host 0.0.0.0 --port 8000

O parâmetro 0.0.0.0 permite acesso externo à aplicação.
