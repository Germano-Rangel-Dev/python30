Dia 25 – Frontend Consumindo API

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

Criar uma API é apenas metade do caminho. Para que usuários interajam com o sistema, precisamos de um frontend que consuma essa API, exiba dados e envie informações.

Neste dia, você aprenderá como usar HTML e JavaScript para se comunicar com uma API Python criada com FastAPI.

Conceito-chave:
O frontend consome a API através de requisições HTTP feitas com JavaScript.

2️⃣ Comunicação Frontend ↔ Backend

No JavaScript moderno, utilizamos a função fetch() para fazer requisições HTTP para APIs.

  • Buscar dados (GET)
  • Enviar dados (POST)
  • Atualizar dados (PUT)
  • Excluir dados (DELETE)

3️⃣ Exemplo Prático – Consumindo API

Exemplo 1 – HTML básico


<button onclick="carregarUsuarios()">Carregar Usuários</button>
<ul id="lista"></ul>

Exemplo 2 – JavaScript com fetch()


function carregarUsuarios() {
  fetch("http://127.0.0.1:8000/usuarios")
    .then(resposta => resposta.json())
    .then(dados => {
      const lista = document.getElementById("lista");
      lista.innerHTML = "";
      dados.forEach(u => {
        const li = document.createElement("li");
        li.textContent = u;
        lista.appendChild(li);
      });
    });
}

Esse código busca dados da API e exibe no HTML.

4️⃣ Enviando Dados para a API


fetch("http://127.0.0.1:8000/usuarios", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    nome: "Germano",
    email: "email@email.com"
  })
})

Aqui enviamos dados do frontend para o backend.
