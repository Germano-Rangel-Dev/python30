Dia 26 – Autenticação com Token (JWT)

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

Nos dias anteriores, você implementou login tradicional baseado em sessão. Porém, aplicações modernas — especialmente APIs — utilizam tokens para autenticação.

O padrão mais utilizado é o JWT (JSON Web Token), que permite autenticar usuários de forma segura, escalável e sem depender de sessões no servidor.

Conceito-chave:
JWT é um token assinado que representa a identidade do usuário.

2️⃣ Como funciona o JWT

O fluxo de autenticação com JWT é simples:

  1. Usuário envia login e senha
  2. Servidor valida credenciais
  3. Servidor gera um token JWT
  4. Frontend guarda o token
  5. Token é enviado em cada requisição protegida

O token contém informações codificadas e assinadas, impedindo alterações.

Passo 2 – Gerando token JWT


from jose import jwt
from datetime import datetime, timedelta

SECRET_KEY = "segredo_super_secreto"
ALGORITHM = "HS256"

def criar_token(dados: dict):
    dados = dados.copy()
    expiracao = datetime.utcnow() + timedelta(minutes=30)
    dados.update({"exp": expiracao})
    return jwt.encode(dados, SECRET_KEY, algorithm=ALGORITHM)

Passo 3 – Login retornando token


@app.post("/login")
def login(usuario: UsuarioLogin):
    if autenticar(usuario.email, usuario.senha):
        token = criar_token({"sub": usuario.email})
        return {"access_token": token, "token_type": "bearer"}
    return {"erro": "Credenciais inválidas"}

4️⃣ Protegendo Rotas


from fastapi import Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="login")

def verificar_token(token: str = Depends(oauth2_scheme)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        return payload
    except:
        raise HTTPException(status_code=401, detail="Token inválido")


@app.get("/protegido")
def rota_protegida(usuario = Depends(verificar_token)):
    return {"mensagem": "Acesso autorizado"}
