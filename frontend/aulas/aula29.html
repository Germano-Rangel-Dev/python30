Dia 29 – Boas Práticas e Clean Code

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

Saber fazer um código funcionar não é suficiente no mercado profissional. É necessário que o código seja legível, organizado, manutenível e escalável.

Boas práticas e Clean Code são um conjunto de princípios que tornam o código mais fácil de entender, modificar e evoluir ao longo do tempo.

Conceito-chave:
Clean Code é escrever código que outras pessoas (e você no futuro) consigam entender facilmente.

2️⃣ Princípios Fundamentais de Clean Code

  • Nomes claros e descritivos
  • Funções pequenas e objetivas
  • Evitar duplicação de código
  • Organização em módulos
  • Comentários apenas quando necessário

3️⃣ Exemplos Práticos

❌ Código ruim


def f(a,b):
    return a+b

✅ Código limpo


def somar_dois_numeros(numero1, numero2):
    return numero1 + numero2

Nomes claros tornam o código autoexplicativo.

4️⃣ Organização de Projetos Python

Projetos profissionais seguem uma estrutura organizada: 


projeto/
 ├── app/
 │   ├── main.py
 │   ├── models.py
 │   ├── services.py
 │   └── routes.py
 ├── tests/
 ├── requirements.txt
 └── README.md

5️⃣ Comentários e Documentação

Bons códigos precisam de poucos comentários, pois são autoexplicativos. Quando necessário, utilize docstrings. 


def calcular_media(valores):
    """
    Calcula a média de uma lista de números.
    """
    return sum(valores) / len(valores)
