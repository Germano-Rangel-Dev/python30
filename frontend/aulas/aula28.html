Dia 28 – Testes Automatizados em Python

Curso Python em 30 Dias

1️⃣ Introdução Conceitual

À medida que aplicações crescem, testar manualmente cada funcionalidade se torna impraticável e sujeito a erros. Para garantir qualidade, estabilidade e confiabilidade, utilizamos testes automatizados.

Testes automatizados permitem validar automaticamente se o código continua funcionando corretamente após alterações, evitando regressões.

Conceito-chave:
Testes automatizados verificam automaticamente se o código funciona como esperado.

2️⃣ Tipos de Testes

  • Teste unitário – testa pequenas partes do código
  • Teste de integração – testa a integração entre módulos
  • Teste de sistema – testa o sistema completo

Neste curso, focamos principalmente em testes unitários.

3️⃣ Ferramentas de Teste em Python

Python possui ferramentas nativas e bibliotecas externas para testes:

  • unittest – biblioteca padrão do Python
  • pytest – biblioteca moderna e muito utilizada

4️⃣ Exemplo Prático com unittest

Função a ser testada


def soma(a, b):
    return a + b

Teste unitário


import unittest
from calculos import soma

class TestCalculos(unittest.TestCase):
    def test_soma(self):
        self.assertEqual(soma(2, 3), 5)

if __name__ == "__main__":
    unittest.main()

O teste verifica automaticamente se a função retorna o valor correto.

5️⃣ Executando os Testes


python -m unittest

Se tudo estiver correto, o teste passa. Caso contrário, o erro é exibido.
